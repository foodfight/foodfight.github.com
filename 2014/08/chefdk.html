Show Notes

Here’s a brief outline of some of the things we’d like to discuss.

  • Intro to ChefDK

    • The Chef Development Kit (ChefDK)
    • What’s in the box?
      • Berkshelf 3.0
      • Test Kitchen
      • ChefSpec
      • Foodcritic
      • All the Chef tools you’d usually install on a workstation: knife, ohai, chef-client, chef-zero
      • New command-line tool called chef
    • Supported operating systems - As of version 0.2.1 the following Operating Systems are supported
      • Mac OS X 10.8, 10.9
      • Windows 7, 8, 8.1 & Server 2008 R2, 2012, 2012 R2
      • Ubuntu Linux 12.04, 13.10
      • Red Hat Enterprise Linux 6

  • Using ChefDK

    • Using the chef command
      • verify
      • Some glue commands - exec, gem, and shell-init
      • generate
        • coobkook
        • recipe
        • attribute
        • template
        • file
        • lwrp
        • app
    • Generators
      • what does each generator create?
      • how do they work?
      • how do I create my own?

  • Future of ChefDK

    • Policyfile.rb
  • Chef Docs Repo: https://github.com/opscode/chef-docs

  • Questions for the panelists:

    • should the CI server use ChefDK or should the cookbook’s Gemfile specify the version of testing tools?
    • Is ChefDK ready for me to start using today? What edges am I likely to find?
    • Should I consider migrating my knife plugin to a chef plugin?
    • When can I recommend ChefDK to Windows engineers?
    • Is there any way to configure ChefDK, like generator cookbook path

  • Commands to install ChefDK from command line:

    • Install latest released version - curl -L https://www.getchef.com/chef/install.sh | sudo bash -s -- -P chefdk
    • Install nightly build - curl -L https://www.getchef.com/chef/install.sh | sudo bash -s -- -P chefdk -n
